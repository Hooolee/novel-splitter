# 开发思维指南

> **目的**: 在编码前系统性地思考，在问题变成 bug 之前发现它们。
>
> **核心理念**: 30 分钟的思考能节省 3 小时的调试。

## 可用的思维指南

| 指南 | 目的 | 何时使用 |
|------|------|----------|
| [跨层思维](./cross-layer-thinking-guide.md) | 思考前端-后端-Tauri 层之间的数据流 | 实现跨越 2+ 层的功能前 |
| [实现前检查清单](./pre-implementation-checklist.md) | 验证编码前的准备工作 | 开始任何功能实现前 |

## 核心原则

1. **搜索再写** - 创建新模式前先搜索现有模式
2. **思考再编码** - 5 分钟检查清单节省 50 分钟调试
3. **文档化假设** - 让隐式假设变得明确
4. **多层验证** - 变更通常需要在多处更新
5. **从 bug 中学习** - 修复非琐碎 bug 后，将经验添加到指南中

**语言**: 所有文档使用**中文**编写。
