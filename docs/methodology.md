# 拆书方法论

这份方法论对应当前 V2 主线，强调两件事：

1. 先用数据库和榜单看大势
2. 再用 4 Agent 拆出可复用细节

## 宏观扫榜 vs 微观拆书

### 宏观扫榜

宏观层关注的是题材和赛道：

- 哪些标签在当前样本里更集中
- 哪些书在榜单变化上更靠前
- 哪些标签的平均共识更负向
- 哪些赛道值得继续追踪

宏观层输出的是趋势，不是单本结论。

### 微观拆书

微观层关注的是单本小说可复用的方法：

- 开篇如何起钩子
- 主角是什么人设
- 金手指是什么类型
- 章节结尾怎么留钩子
- 节奏是密还是松
- 有哪些写法值得抄作业

微观层输出的是方法，不是市场分布。

## 4 Agent 解耦

当前评估不是一组 Agent 重复填表，而是分工：

- `reader`：普通读者视角，判断追读欲望
- `editor`：编辑视角，判断签约和推荐价值
- `author`：作者视角，判断写作技法是否成立
- `analyst`：客观拆书，输出事实字段

这样做的目的是：

- 避免重复浪费 token
- 避免主观判断污染客观字段
- 避免字段之间互相抢职责

## `consensus` 规则

当前共识只由前三个主观 Agent 决定，分为 5 态：

- `all_yes`
- `majority_yes`
- `divergent`
- `majority_no`
- `all_no`

中文显示时必须映射为人能直接读懂的标签，不能直接暴露英文枚举。

## `breakdown` 设计原则

`breakdown` 是客观拆书字段，不参与投票。

### 7 个字段

- `goldfinger_type`：金手指类型
- `protagonist_archetype`：主角人设
- `opening_hook`：开篇钩子说明
- `hook_density`：钩子密度
- `pacing_notes`：节奏说明
- `chapter_end_hook_types`：章节结尾钩子类型
- `learning_points`：可借鉴写法

### 设计要求

- 字段可以缺失，但不能硬编成伪事实
- 列表字段要兼容偶发的字符串返回
- 前端占位卡在缺字段时要自然退化，不崩溃

## 7 维洞察

当前 `选题雷达` 顶部的 7 维洞察来源如下：

1. 热门题材
2. 关注点
3. 共识
4. 高频金手指
5. 主角人设
6. 黑马
7. 风险赛道

这 7 维分别回答的是：

- 题材热度
- 读者/编辑/作者分别在意什么
- 市场整体态度如何
- 爆点和设定类型的集中度
- 主角类型的集中度
- 榜单变化最快的是谁
- 哪些标签平均共识更负向

## 方法边界

- 不把宏观趋势当作单本结论
- 不把主观投票混进客观字段
- 不把一次评估结果当成永久定论
- 不用文件流解释当前主路径
